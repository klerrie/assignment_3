CLOUD SERVICES AND INFRASTRUCTURE

Overview of cloud services and how to use them.

CLOUD PROVIDERS
- AWS: Primary cloud provider
- Azure: Microsoft services integration
- GCP: Google Cloud for specific projects
- Multi-cloud: Strategy for redundancy
- Management: CloudOps team manages infrastructure

STORAGE SERVICES
- S3: Object storage on AWS
- OneDrive: Personal file storage
- SharePoint: Team document storage
- Limits: 1TB per user on OneDrive
- Backup: Automatic backup to cloud

COMPUTE RESOURCES
- EC2: Virtual servers on AWS
- Lambda: Serverless functions
- Containers: Docker and Kubernetes
- Scaling: Auto-scaling configured
- Monitoring: CloudWatch for metrics

DATABASE SERVICES
- RDS: Managed relational databases
- DynamoDB: NoSQL database
- Redshift: Data warehouse
- Access: Through approved tools only
- Backup: Automated daily backups

NETWORKING
- VPC: Virtual private clouds
- VPN: Site-to-site connections
- CDN: CloudFront for content delivery
- DNS: Route 53 for domain management
- Security: Security groups and NACLs

MONITORING AND LOGGING
- CloudWatch: AWS monitoring
- Azure Monitor: Azure monitoring
- Logs: Centralized logging system
- Alerts: Automated alerting configured
- Dashboards: Custom dashboards available

COST MANAGEMENT
- Budgets: Set for each project
- Alerts: Cost threshold alerts
- Optimization: Regular cost reviews
- Reserved instances: For predictable workloads
- Reporting: Monthly cost reports

SECURITY AND COMPLIANCE
- IAM: Identity and access management
- Encryption: Data encrypted at rest and in transit
- Compliance: SOC 2, ISO 27001 certified
- Auditing: Regular security audits
- Policies: Cloud security policies enforced
