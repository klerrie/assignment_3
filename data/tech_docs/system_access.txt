SYSTEM ACCESS AND ACCOUNT MANAGEMENT

How to request and manage access to company systems.

NEW EMPLOYEE SETUP
- Account creation: Automated on start date
- Email: Created within first day
- Systems: Access granted based on role
- Training: IT orientation in first week
- Equipment: Laptop and peripherals provided

ACCESS REQUESTS
- Portal: Submit requests through IT portal
- Approval: Manager approval required
- Processing: 1-2 business days
- Notification: Email confirmation when granted
- Documentation: Access documented in system

COMMON SYSTEMS
- HRIS: Employee self-service portal
- CRM: Salesforce for sales teams
- ERP: SAP for operations
- Finance: QuickBooks for accounting
- Project tools: Jira, Confluence access

ROLE-BASED ACCESS
- Standard user: Basic system access
- Department access: Based on job function
- Admin access: Requires additional approval
- Developer access: Code repository and tools
- Executive access: Reporting and analytics

ACCESS REVIEWS
- Quarterly: Regular access audits
- Offboarding: Immediate revocation
- Role changes: Access updated automatically
- Inactive accounts: Disabled after 90 days
- Compliance: Annual access certification

TROUBLESHOOTING ACCESS
- Login issues: Reset password first
- Missing access: Submit access request
- Permission errors: Contact IT support
- Account locked: Wait 30 minutes or contact IT
- MFA issues: Use backup codes or contact IT

REMOTE ACCESS
- VPN: Required for remote connections
- Setup: Instructions in IT portal
- Support: Available 24/7 for VPN issues
- Security: MFA required for VPN
- Performance: Contact IT for speed issues
